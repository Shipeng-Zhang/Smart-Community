# 实验报告补充

## 一、项目主题

**智慧社区居家安全与能耗协同管理系统**

系统聚焦老旧社区治理中最常见、最有现实价值的几个问题：

1. 厨房燃气泄漏与烟雾异常  
2. 用电过载与高峰时段能耗浪费  
3. 漏水隐患与财产损失风险  
4. 老人户低活动异常与主动关怀需求  

相比单纯展示数据采集，本项目更强调社区治理闭环和智慧物业运维效率，更贴近真实应用场景。

## 二、针对常见智慧社区系统缺陷的创新优化

### 创新点 1：多源告警关联闭环

### 对应常见缺陷

传统智慧社区系统常常存在：

- 每类传感器各自告警
- 告警之间缺少关联
- 告警后缺少持续跟踪和处置闭环

### 本项目优化

本系统将燃气、烟雾、用电、漏水等多源信号统一纳入同一住户风险上下文：

1. 一旦同一住户同时出现两类以上风险信号，系统将其视为关联风险住户。  
2. 系统自动生成工单并进入工单流转。  
3. 若处理超时，系统自动升级工单优先级。  
4. 恢复稳定后，工单自动进入闭环流程。  

### 创新价值

实现了从“单点告警”到“关联治理”的升级，避免系统只会报警却无法有效处置。

## 创新点 2：分时能耗画像与峰值抑制

### 对应常见缺陷

很多社区系统只能展示总能耗或累计用电量，无法回答：

- 哪些住户在晚高峰负载异常？
- 哪些住户存在持续高功率浪费？
- 物业如何做分时节能管理？

### 本项目优化

系统实时计算：

1. 当前功率  
2. 功率波动量  
3. 分时趋势  
4. 高峰负载住户数量

并在前端中增加：

- 风险与能耗趋势图  
- 告警类型分布图  
- 设备页筛选与排序  

### 创新价值

这使系统从“安全监测系统”扩展为“安全 + 节能”的复合型社区管理平台。

## 创新点 3：弱势住户主动关怀

### 对应常见缺陷

传统智慧社区系统更关注设备，却较少体现对人的服务，尤其忽视：

- 独居老人
- 高龄住户
- 长时间低活动异常

### 本项目优化

系统对老人户加入低活动异常检测逻辑：

1. 连续多个采样周期低活动时触发异常识别。  
2. 自动生成“老人关怀”工单。  
3. 引导网格服务组或物业人员主动回访。  

### 创新价值

让智慧社区系统不仅“会管设备”，也“会关怀居民”，更符合实际社区治理的人本需求。

## 三、界面与交互优化

### 1. 趋势图

旧版本存在横坐标时间标签堆叠问题。  
新版本改为：

- 抽样显示时间刻度
- 保留关键点和终点标签
- 提升趋势可读性

### 2. 数据图

在“风险与能耗趋势”图下方新增：

**告警类型分布图**

用于更清楚地展示当前风险结构，而不是仅看一张折线图。

### 3. 工单区域重新排版

旧版工单流转区域存在卡片拥挤、信息密度过高的问题。  
新版调整为：

- 左侧横向看板
- 右侧工单详情面板
- 卡片只展示关键信息
- 动作按钮集中放入详情区

整体更接近 OA 系统的操作逻辑。

### 4. 多页面切换

系统不再是单页堆砌，而是拆分为：

1. 总览  
2. 工单  
3. 告警  
4. 设备  
5. 创新  
6. 设置
